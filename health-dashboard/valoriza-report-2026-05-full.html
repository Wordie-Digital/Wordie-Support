Hero image and render-blocking assets are slowing mobile load
Valoriza scores 68/100 on Google's mobile performance test (PageSpeed Insights lab data, run 7 May 2026). The critical issue is Largest Contentful Paint of 8.3 seconds — Google's threshold for a good LCP is 2.5 seconds. The cause is a combination of 6 render-blocking CSS/JS requests that delay the browser from rendering anything, and a large hero image that is not preloaded or properly optimised. Real-user field data shows better numbers (LCP 1.3s), but Google's ranking algorithm uses lab scores, so the 68/100 represents the ranking risk.

Business risk

An LCP of 8.3s is a direct Google ranking signal. Service and contact pages competing for organic search traffic will be penalised relative to faster competitors. The fix is targeted and achievable in a single sprint.

Recommended action

Preload the hero image using , convert it to WebP format, and defer the 6 render-blocking CSS/JS files. Expected outcome: LCP under 3s, PageSpeed score 80+.

Recommended actions

Prioritised list of actions for this month

HIGH PRIORITY
Preload hero image and convert to WebP format
The hero image is the LCP element — preloading it and serving it in WebP format is the single highest-impact change. Expected LCP improvement from 8.3s to under 3s, lifting PageSpeed score to 80+. This directly affects Google rankings for Valoriza's service pages.
HIGH PRIORITY
Defer 6 render-blocking CSS and JavaScript requests
Render-blocking resources are preventing the browser from displaying any content until they finish loading. Deferring non-critical JS and inlining critical CSS will reduce First Contentful Paint and Speed Index significantly.
MEDIUM PRIORITY
Apply all pending plugin and theme updates
Establishing a clean update baseline in the first maintenance cycle removes known vulnerability exposure and ensures compatibility with the latest WordPress core.
MEDIUM PRIORITY
Remove unused JavaScript and CSS
Google flags unused JS and CSS as contributing to unnecessary page weight. Removing or conditionally loading these assets will reduce overall load time and improve the PageSpeed score.
